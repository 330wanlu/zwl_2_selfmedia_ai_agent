## 2. 运营人员使用的扭转流程（前端视角）

```
打开 http://127.0.0.1:5173/tasks
        ↓
① 输入内容方向 →「创建并开始」
        ↓ 自动进入详情页，轮询等待
② 选题决策区：单选卡片 →「确认选题」（或换一批）
        ↓ AI 写文案…
③ 文案审核区：阅读正文 →「通过」或填意见「提交修改意见并重写」
        ↓ AI 分镜 + 出图…
④ 图片审核区：预览 → 勾选重绘 /「全部通过」
        ↓ AI 生成内容包…
⑤「查看内容包」→ 复制文字、下载图片、标记已发布
```

中间产物与后端阶段 1/2 相同（topics / content_versions / images / publish_records）；前端只负责展示与提交决策。

## 6. 遇到的问题与解决

### 问题 1：后端未启动时无法联调

- **现象**：开始阶段 3 时 `GET /health` 不通。
- **处理**：自测前先拉起  
  `uv run uvicorn app.main:app --reload --reload-dir app --port 8000`  
  与 `npm run dev -- --host 127.0.0.1 --port 5173`。

### 问题 2：E2E 误匹配「流水线进度」里的步骤文案

- **现象**：`getByText('图片审核')` / `.ant-card` + `hasText: '文案审核'` 先命中 Steps，导致「按钮找不到」或「假阳性已进入图片审核」。
- **解决**：决策区定位改为 **卡片标题**  
  `.ant-card` + `.ant-card-head-title` 精确匹配「文案审核 / 图片审核 / 选题决策」。

### 问题 3：Ant Design 双字按钮插入空格

- **现象**：按钮文案实际是 `通 过`（中间空格），`hasText: /^通过$/` 匹配失败。
- **解决**：
  1. E2E 用 `/通\s*过/`；
  2. 全局 `ConfigProvider` 设置 `button={{ autoInsertSpace: false }}`，避免运营界面与自动化再踩坑。

### 问题 4：出图等待时间长

- **现象**：4 张图串行约 1.5～2 分钟；E2E 若过早断言图片卡片会超时。
- **解决**：图片卡片 `waitFor` 超时设为 600s；支持 `RESUME_FROM=images` 续跑。

### 问题 5：PowerShell 不支持 `&&`

- **现象**：`npm install && npm install antd ...` 报语法错误。
- **解决**：改用 `;` 分号串联，或分开执行。

## 7. 本地运行方式

```powershell
# 终端 1：后端
cd backend
uv run uvicorn app.main:app --reload --reload-dir app --port 8000

# 终端 2：前端
cd frontend
npm install          # 首次
npm run dev -- --host 127.0.0.1 --port 5173
```

- 前端：http://127.0.0.1:5173  
- 后端 / Swagger：http://127.0.0.1:8000/docs  
- 开发代理：浏览器请求 `/api/*`、`/images/*` 由 Vite 转到 8000，无需配 CORS 也能用。
